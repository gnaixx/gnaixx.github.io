0x01 Load 流程

想要重写 DexClassLoader 先要弄清楚它的 Load 流程(以下代码以6.0为例)。用 DexClassLoader 进行 dex 文件的 Load, 主要用到了两个函数：

上面操作可以分为两部分：1.dex文件加载 2.class类加载。可以分析这个两个步骤的流程。

dex 文件加载

从 DexClassLoader.java 开始，该类继承于 BaseClassLoader，内部只实现了一个构造函数，并且直接调用父类的构造函数(以下源码去掉了部分注释)。
DexClassLoader.java:

BaseDexClassLoader 构造函数创建了一个 DexPathList 对象，主要是负责解析加载的 DEX 文件的解析。
BaseDexClassLoader.java:

DexPathList 构造函数中调用了 makePathElements 解析了 dex 文件。
DexPathList.java:

makeDexElements 中调用 loadDexFile 加载文件，并返回一个 DexFile 对象。
DexPathList.java:

loadDexFile 中如果不存在 optimizedDirectory 直接 new 一个 DexFile，如果存在则调用 DexFile.loadDex 方法返回一个 DexFile 对象。
DexPathList.java:

DexFile 中有多个构造函数，上面DexPathList.loadDexFile 调用了其中两个，可以看到最后关键的代码都是 openDexFile 函数。
DexFile.java:

openDexFile 是 Load dex 文件的关键，他的实现很简单只调用了一个 native 方法 openDexFileNative 这个方法不同版本的 API 实现都有差别，下一章节会介绍到。
DexFile.java:

到这里 dex 文件的加载 java 层的流程基本就结束了。

class 类加载

DexClassLoader 加载一个类是通过 loadClass 方法，但是在 DexClassLoader 和 BaseDexClassLoader 中都没有该方法，该方法的实现是在 ClassLoader 中。loadClass 中有三处返回，1.判断 BootClassLoader 是否加载 2.父类ClassLoader是否加载 3.通过findClass加载。
ClassLoader.java:

ClassLoader.findClass 实现是空的，真正的实现在 BaseDexClassLoader 中，调用了 DexPathList 的 findClass 方法。
BaseDexClassLoader.java:

DexPathList 通过遍历所有加载的 DexFile 对象，调用 DexFile 的 loadClassBinaryName 方法。
DexPathList.java:

loadClassBinaryName 方法直接调用了 defineClass 方法，而 defineClass 调用了一个 native 方法 defineClassNative。其中第三个参数 cookie 即方法 openDexFile 的返回值。
